# 第14章：AI API怎么配置——从获取Key到安全使用

> 本章解决一个问题：很多人想用AI做自己的项目，但卡在第一步——不知道怎么配置API。这一章手把手教你。

很多人听说可以用API调用AI来做自己的应用，但一查文档就懵了。什么是API Key？怎么获取？在哪配置？安全吗？多少钱？这本章帮你解决这些问题。

读完本章，你会知道主流AI的API怎么申请、怎么在代码里配置、怎么省钱、怎么保证安全。

---

## 一、什么是API和API Key

首先澄清一个概念很多人混淆的事：API和API Key不是一回事。

API是应用程序接口的缩写。你可以把它理解为软件之间对话的方式。你要让自己的程序调用AI的服务，就需要通过API来"打招呼"。API Key则是这个"打招呼"的口令。每个注册用户有一个唯一的Key，相当于你在AI服务平台的身份证。

没有Key，AI平台不知道谁在调用他们的服务，也没法统计用量和收费。所以第一步，你要去AI平台申请一个Key。

---

## 二、主流AI平台API怎么申请

国内和国外的AI平台申请API Key的流程不太一样。

**国内平台。**

豆包在火山引擎开放平台申请API Key。注册账号后进入控制台，创建应用后会获得AppID和API Key。流程比较简洁，有免费额度。

Kimi在月之暗面开发者平台申请。注册后进入开发者后台，创建API Key即可。近期才开始开放API申请，可能需要申请排队。

通义千问在阿里云百炼平台申请。阿里云的流程相对正式，需要创建阿里云账号、开通百炼服务，然后获取API Key。

文心一言在百度智能云开放平台申请。同样需要注册百度账号、开通服务，然后获取Key。

**国外平台。**

ChatGPT的API在OpenAI平台申请。注册账号后进入API Keys页面，创建新的Key。OpenAI的API按token收费，新用户有一定免费额度。

Claude的API在Anthropic开发者平台申请。流程类似，创建账号后生成API Key即可。Claude的定价和OpenAI略有不同。

Gemini的API在Google AI Studio申请。Google的AI服务现在也开放了API调用。

**我的建议是：普通人从国内开始。** 国内平台注册简单、支付方便、有免费额度。先用国内API跑通流程，等熟悉了再尝试国外。

## 三、获取Key之后的配置步骤

拿到API Key后，下一步是在代码里配置它。

最简单的方式是通过环境变量。把API Key存在环境变量里，代码运行时读取。这样做的好处是代码里不会直接暴露Key，即使代码泄露，Key也是安全的。

以Python为例，第一步安装SDK。豆包用OpenAI兼容的SDK，通义千问也有官方Python库。第二步配置客户端。以豆包为例，只需要几行代码就能完成配置。

```python
from openai import OpenAI

client = OpenAI(
    api_key="你的API Key",
    base_url="https://ark.cn-beijing.volces.com/api/v3"
)

response = client.chat.completions.create(
    model="doubao-pro-32k",
    messages=[{"role": "user", "content": "你好"}]
)
```

这段代码展示了配置API Key的基本逻辑。关键点有三个：填入你的Key、选对模型、发送正确的请求格式。官方文档里都有示例代码，复制过来改一下Key就能跑。

## 四、怎么省钱——API费用优化

很多人怕API贵，其实用对了方法，费用可以控制得很低。

**第一个方法：选对模型。**

同一个平台有多个模型，价格差很远。比如OpenAI的GPT-4比GPT-3.5贵几十倍。很多场景不需要那么强的模型，用便宜的模型就够。豆包的Pro版比Lite版贵，但Lite版已经能应付大多数日常需求。先用便宜的模型测试，效果够用就不要升 级。

**第二个方法：控制请求量。**

API是按调用次数和Token数量收费的。一个技巧是把多次请求合并。比如你有一堆问题要问AI，不要逐个调用，而是把问题整理成一批，用一次请求发过去。这样只收一次调用的费用。

**第三个方法：用缓存。**

相同的问题不需要反复调用AI。把常见的问答结果存起来，下次有人问直接返回缓存。只有缓存里没有的才调用API。

**第四个方法：设置用量限制。**

大多数平台可以设置每月最高消费限额。设置为五十元或一百元，即使出了bug也最多花这么多钱。这是防止意外开支的最有效方式。

对于普通人来说，每个月花几十元到一百元已经完全够用了。不要被"AI很贵"的印象吓到，那是因为很多人没控制好用量。

## 五、API安全——怎么保护你的Key

API Key就是你的账号密码。泄露了不仅可能被滥用，还可能产生额外费用。

**第一个原则：不要把Key写在代码里。**

很多新手直接把Key粘贴在代码里，然后上传到GitHub。这样做Key很快就被人偷走了。正确做法是用环境变量或配置文件。配置文件要加入gitignore，不要提交到代码仓库。

**第二个原则：最小权限。**

申请API Key时，很多平台可以选择权限范围。只开你需要的权限，不要开管理员权限。如果只是聊天，就不要开模型训练的权限。

**第三个原则：定期更换。**

Key跟密码一样，用久了可能泄露。隔几个月换一个Key不是什么麻烦事，但能提高安全性。

**第四个原则：用完就删。**

如果你在某个环境里不再使用某个Key，去平台把它删除。很多时候Key泄露是因为项目不做了，但Key还在平台上挂着。

## 六、一个配置示例

我用一个具体例子来说明完整流程。

假设你用Python写了一个自动回复邮件的小工具，需要调用通义千问的API来生成回复内容。

第一步去阿里云百炼平台注册账号、开通服务、创建一个API Key，记下来。第二步在本地安装阿里云的SDK：pip install dashscope。第三步编写代码，从环境变量读取Key，配置客户端，发送请求。

代码大约二十行就能跑起来。跑通之后，你可以把这个工具部署到服务器上，让它自动处理邮件。但要注意在部署环境里也要配置好环境变量，不要把Key硬编码。

这就是一个典型的"配置API并用来做项目"的完整流程。很多AI应用都是这个套路：获取Key、配置SDK、调用服务、处理结果。

## 本章小结

配置AI API的核心流程是：申请Key、安装SDK、编写代码。成本控制的关键是选对模型、控制请求量、用缓存和设限额。安全方面不要把Key写在代码里，用环境变量、最小权限、定期更换。这件事没那么难，第一次配置可能花半小时，熟悉之后十分钟就能搞定。任何想用AI做自己项目的人，这一关早晚要过。
