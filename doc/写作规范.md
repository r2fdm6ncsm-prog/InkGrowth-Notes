# 写作风格指南

> 本书的风格指南，而非一份"禁止清单"。
> 它定义的是：**我们追求什么，以及为什么这样追求。**

---

## 一、一句话风格定义

**吴军的理性深度 + 得到系的结构化表达 = 本书的写作风格。**

- **吴军风格**：每句话都有信息密度，不做无意义铺陈。读者能感受到作者的判断力和思考深度。
- **得到系风格**：结构化、模块化、可学习。读者能轻松跟上，读完后能记住、能转述。

两者结合，保证内容质量的同时兼顾阅读体验。

---

## 二、目标读者决定了语气上限

本书读者是**中国普通人**：

| 特征 | 含义 |
|------|------|
| 非技术背景 | 不要用专业术语定义专业术语 |
| 时间碎片化 | 每段都要让读者有所收获 |
| AI 焦虑 | 语气要温和，不要制造更多焦虑 |
| 渴望可操作性 | 不要只讲道理，要给出路 |

---

## 三、句子级别的写作原则

1. **一个句子一个信息** — "Prompt 的本质不是提问，而是任务设计。"（而不是写成绕来绕去的长句）
2. **优先使用准确动词** — "把工具分成三类"（而不是"对工具进行分类归纳整理"）
3. **长短句交替制造节奏** — 短句密集段落用于强调和结论，长句用于展开推导和举例
4. **减少低质量连接词** — "总而言之""综上所述"尽量不用，用自然过渡代替

---

## 四、段落级别的写作原则

**一个段落一个功能**。每个段落只做一件事：定义概念、解释原理、给出案例、展开推导、提供方法或总结规律。同一段不混合多个功能。

**段落之间要有递进感**。不是列清单，而是带着读者往前走。一段没读完，读者想知道下一段说什么。

---

## 五、逻辑结构优先级

| 优先级 | 结构 | 适用场景 |
|--------|------|----------|
| 1 | 结论 → 原因 → 示例 | 观点性内容（最推荐） |
| 2 | 问题 → 分析 → 方案 | 解决型内容 |
| 3 | 概念 → 原理 → 应用 | 知识型内容 |
| 4 | 现象 → 本质 → 推演 | 洞察型内容 |

**不绕圈。** 读者不需要知道你是怎么想到的，只需要知道你想让他知道的。

---

## 六、内容质量要求

**必须达到的标准**：
- 实际价值 — 读者读完能带走一个确定的认知或方法
- 可执行 — 不只是"知道"，还要"能做"
- 可学习 — 结构清晰，能拆解、能回顾
- 可复用 — 方法论不依赖特定工具
- 可沉淀 — 不是一次性阅读，而是长期可查阅的参考

**必须避免的内容**：
- 空泛观点："AI 很重要" → 改成 "AI 正在重新定义'能力'这个词的含义"
- 模糊建议："多练习" → 改成 "每天花 15 分钟做一次 Prompt 拆解练习"
- 纯情绪输出：没有信息量的感叹和抒情

---

## 七、差异化定位

| 我们不是 | 我们是 |
|----------|--------|
| 工具说明书 | 认知升级路线 |
| 编程教材 | 底层逻辑讲解 |
| Prompt 合集 | 方法论体系 |
| AI 新闻解读 | 可复用的思维框架 |

---

## 八、最终交付标准

一篇合格的书稿章节，应该：

1. 读者看完能**用自己的话转述核心观点**
2. 读者能**立刻做一件事**（哪怕很小）
3. 读者愿意**分享给朋友**
4. 编辑不需要**大段重写**
