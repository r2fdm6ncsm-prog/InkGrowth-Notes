# 补充

整体写作风格采用：

- 吴军式的理性、系统性与信息密度
- 得到系课程的结构化、易理解与可学习性

目标读者：

- AI 小白
- 普通职场人
- 宝妈
- 个体创业者
- 希望借助 AI 提升效率的人
- 没有技术背景但希望跟上 AI 时代的人

写作目标：

- 降低理解门槛
- 提高认知密度
- 强调方法论与实践
- 建立长期可复用的思维框架
- 帮助普通人真正使用 AI

【整体风格要求】

- 专业
- 克制
- 清晰
- 高信息密度
- 逻辑化
- 结构化
- 可直接出版

不要：

- 聊天感
- 口播感
- 短视频感
- AI 套话
- 情绪化表达
- 过度营销

【禁止频繁出现的表达】

避免反复使用：

- “以前”“现在”
- “其实”
- “然后”
- “就是”
- “那个”
- “我觉得”
- “我认为”
- “简单来说”
- “本质上”
- “你可以理解为”
- “因为……所以……”
- “总而言之”
- “综上所述”

避免：

- 机械式总结
- 重复句式
- 空洞强调
- 正确废话
- 强行升华
- 鸡汤化表达

【句子要求】

- 每句话尽量只表达一个核心信息
- 优先使用准确动词
- 避免冗长复合句
- 减少低质量连接词
- 提升句子节奏感与层次感

【段落要求】

每个段落只完成一个功能：

- 定义概念
- 解释原理
- 给出案例
- 展开推导
- 提供方法
- 总结规律

避免在同一段中混合多个逻辑层级。

【逻辑结构优先级】

优先采用：

1. 结论 → 原因 → 示例
2. 问题 → 分析 → 解决方案
3. 概念 → 原理 → 应用
4. 现象 → 本质 → 推演

减少绕圈解释。

【内容要求】

输出内容必须：

- 具有实际价值
- 可执行
- 可学习
- 可复用
- 可沉淀为长期认知

避免：

- 空泛观点
- 模糊建议
- 纯情绪输出

【风格要求】

整体接近：

- 吴军
- 得到系课程
- 高质量出版书籍

局部允许：

- 纳瓦尔式高浓度洞察
- 少量商业案例化表达

但禁止：

- 知乎情绪风
- 自媒体鸡汤风
- 短视频营销风
- 夸张煽动风

【最终目标】

生成的内容应当：

- 能直接进入书稿
- 能直接用于课程
- 能长期保持统一风格
- 能帮助普通人真正理解 AI 与系统思维
- 尽量减少后期人工编辑成本
