# 项目目录结构规范

## 一、根目录

```
InkGrowth-Notes/
│
├── index.html        # GitHub Pages 入口，302 跳转 → prototype/index.html
├── 404.html          # GitHub Pages 404 页面，跳转 → prototype/index.html
├── README.md         # 项目说明
├── AGENTS.md         # Agent 交接文档（仅 AI Agent 读取）
├── .gitignore        # Git 忽略规则
│
├── doc/              # 项目文档
├── manuscript/       # 书稿原文（Markdown）
├── prototype/        # 网站原型（HTML）
├── archive/          # 存档
├── project/          # 软件项目（当前不适用）
├── database/         # 数据库脚本（当前不适用）
└── utils/            # 工具脚本
```

## 二、各目录职责

### doc/ — 项目文档
所有项目相关文档集中在此。包括但不限于：
- `PRD.md` — 产品需求文档
- `作者人格.md` — 作者语气/人格定义
- `写作规范.md` — 写作风格指南
- `成长路径.md` — 全书大纲与章节结构
- `QClaw写作指令.md` — AI写作总指令
- `网站部署与自动更新.md` — 部署说明
- `项目目录规范.md` — 本文件

### manuscript/ — 书稿正文
书稿的 Markdown 源文件，命名规则：
`第{N}章-{英文标题}.md`

示例：
```
第1章-AI时代到底变了什么.md
第2章-AI世界地图.md
```

### prototype/ — 网站原型
静态网站 HTML 文件，当前为单文件 `index.html`。
如果后续扩展为多页面网站，按功能拆分到子目录。

### archive/ — 项目存档
存放已过时的旧版本文档，保留历史记录。

### project/ — 软件项目
当项目需要开发配套软件（如后端API、前端应用）时使用。
- `frontend/` — 前端代码
- `backend/` — 后端代码

### database/ — 数据库脚本
需要数据库时的建表脚本、迁移脚本等。

### utils/ — 工具脚本
项目相关的小工具，如：
- Markdown → HTML 转换脚本
- 内容批量处理脚本
- 备份脚本

## 三、命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文档 | 中文命名，首尾无空格 | `写作规范.md` |
| 书稿 | 第{N}章-{标题}.md | `第1章-AI时代到底变了什么.md` |
| HTML | index.html 或功能英文名 | `index.html` |
| 图片 | 英文或拼音，小写 | `wechat-qr.png` |

## 四、增删文件原则

1. **新增文档** → 放入 `doc/`
2. **新增书稿** → 放入 `manuscript/`
3. **新增HTML** → 放入 `prototype/`
4. **新增工具** → 放入 `utils/`
5. **根目录** 只放索引文件（`index.html`、`404.html`）、项目说明（`README.md`）、Agent指令（`AGENTS.md`）、版本管理（`.gitignore`）
6. **不要** 在根目录堆积文件
7. **删除文件前** 确认不在 `AGENTS.md` 或 `README.md` 中被引用
