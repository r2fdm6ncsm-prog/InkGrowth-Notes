# 网站部署与自动更新说明

## 一、技术栈

| 项 | 内容 |
|---|------|
| 前端框架 | 原生 HTML + CSS + JS（无框架依赖） |
| 网站类型 | 单页应用（SPA），所有内容在 prototype/index.html |
| 托管平台 | GitHub Pages |
| 域名 | https://r2fdm6ncsm-prog.github.io/InkGrowth-Notes/ |
| 自动部署 | GitHub Actions（.github/workflows/deploy.yml） |
| 分支策略 | 直接推送 master，自动部署 |

## 二、网站结构

```
InkGrowth-Notes/
├── index.html                # 根目录，302 跳转到 prototype/index.html
├── 404.html                  # 404 页面，跳转到 prototype/index.html
├── prototype/
│   └── index.html            # 主站（2386行，含所有样式+内容+JS）
└── .github/workflows/
    └── deploy.yml            # GitHub Actions 自动部署
```

### 根目录跳转原理

`index.html` 和 `404.html` 内容一致：
```html
<meta http-equiv="refresh" content="0;url=prototype/index.html">
```

访问 `https://r2fdm6ncsm-prog.github.io/InkGrowth-Notes/` 时：
1. 自动跳转到 `/prototype/index.html`
2. 用户直接访问任何子路径 404 时，同样跳转回主站

## 三、部署流程

### 首次部署
GitHub Pages 配置：
1. GitHub 仓库 Settings → Pages
2. Source: Deploy from a branch
3. Branch: master, / (root)
4. 保存后等待 1-2 分钟部署完成

### 日常更新流程

```
manuscript/*.md (书稿源文件)
        ↓ 手动转换
prototype/index.html (编辑HTML)
        ↓
git add . && git commit -m "xxx"
        ↓
git push origin master
        ↓
GitHub Actions 自动触发部署
        ↓
GitHub Pages 更新 (约1-2分钟生效)
```

### 更新步骤（详细）

```bash
# 1. 编辑 HTML 内容
#    直接修改 prototype/index.html 中的相应章节

# 2. 提交并推送
git add .
git commit -m "feat: 更新第X章内容 / fix: 修复XX问题"
git push origin master

# 3. 等待部署完成
#    可在 GitHub 仓库 Actions 选项卡查看部署进度
```

## 四、GitHub Actions 工作流说明

文件位置：`.github/workflows/deploy.yml`

触发条件：推送 master 分支

执行流程：
1. 检出代码
2. 直接部署到 GitHub Pages（纯静态，无需构建步骤）
3. 部署完成后，网站自动更新

> ⚠️ 如果未来引入构建工具（如 Vite/Webpack），需要在 workflow 中添加构建步骤。

## 五、内容管理

### 内容来源
- 书稿内容：写作输出到 `manuscript/` 目录的 Markdown 文件
- 网站内容：手动将 Markdown 转换为 HTML，更新到 `prototype/index.html`
- 笔记/灵感：直接在 HTML 中编辑

### 内容流转路径（当前）
```
写作 → manuscript/第N章-标题.md
                   ↓ 手动转换
      prototype/index.html (更新相应章节HTML)
                   ↓ git push
      GitHub Pages 自动更新
```

### 优化方向（待实现）

```
manuscript/*.md → 构建工具 → GitHub Pages
```
一次写作、自动转换、自动发布。

## 六、常见问题

### Q: 推送后网站没有更新？
- 检查 GitHub Actions 是否运行成功
- GitHub Pages 部署可能有 1-2 分钟缓存延迟
- 强制刷新（Ctrl+F5）浏览器缓存

### Q: 如何查看部署状态？
- GitHub 仓库 → Actions 选项卡
- 找到最新的 workflow run，点击查看详情

### Q: 部署失败怎么办？
- 检查 Actions 日志中的错误信息
- 确认 `prototype/index.html` 语法正确
- 确认 `.github/workflows/deploy.yml` 配置无误
