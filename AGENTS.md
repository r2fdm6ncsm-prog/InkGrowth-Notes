## 三、写作人格规范（极其重要）

> 完整定义见 `doc/作者人格.md`，以下是快速摘要。
> ⚠️ 注意：网站项目 (AI时代系统思维) 和 书稿项目 (创作生长论) 各有自己的 doc/ 目录。
> **优先使用 创作生长论 中的 doc/ 版本**（更新、更贴合 "生长" 理念）。
> 网站项目中的 doc/ 为早期版本，内容更全面但旧。如有冲突，以创作生长论版本为准。

### 3.1 作者是谁（创作生长论版）

本书由 **23岁大学生作者（Q） + AI写作搭档（Hermes/Crow5）** 协作完成。
核心差异：不是"导师"，不是"前辈"，而是一个**和你差不多年纪、先走了一步的人**。

### 3.2 作者定位（创作生长论版）

> **一个23岁大学生，在AI浪潮里先下水试了试，回过头来给后来者画了一张地图。**
> 语气真诚、轻快、有温度，不给焦虑，给确定感。

### 3.3 语气坐标

| 维度 | 网站版（AI时代系统思维/doc/） | 书稿版（创作生长论/doc/） |
|------|------|------|
| 作者身份 | 30-40岁同行者 | **23岁大学生** |
| 信息密度 | 高 —— 每句话都有信息量 | 中高 —— 不注水，不堆术语 |
| 语气温度 | 温 —— 亲切但不煽情 | **温热 —— 像学长/学姐在跟你聊** |
| 专业程度 | 中高 | 中 |
| 节奏感 | 强 | 轻快 |
| 与读者关系 | "我走过这条路，我把地图给你" | **"我们起点一样，我能走过来你也能"** |

> **写作时以创作生长论版的作者人格为准。** 网站版可作为补充参考。

### 3.4 写作风格一句话

**吴军的理性深度 + 得到系的结构化表达 = 本书写作风格。**

### 3.5 写作红线（绝对不要）

- ❌ 好为人师 — "你必须""你一定要"
- ❌ 居高临下 — "这个问题其实很简单"
- ❌ 技术优越感 — "这都不懂就别学了"
- ❌ 情绪操控 — "再不学就晚了"
- ❌ 自我陶醉 — 大段自我夸奖

### 3.6 每章固定模板

网站版模板（9要素）：
1. 为什么这个问题重要
2. 普通人的错误理解
3. AI时代真正逻辑
4. 一个真实案例
5. 一个流程图/系统图
6. 工具对比（如适合）
7. 小白如何开始
8. 本章一句话总结
9. 本章传播金句

创作生长论版模板（含生长层）：
```
【问题】→ 这一章解决什么问题
【误区】→ 常见的错误认知
【逻辑】→ 核心框架和原理
【案例】→ 真实可操作的例子
【工具】→ 具体用什么、怎么用
【上手】→ 读者可以马上做的事
【总结】→ 核心要点
【金句】→ 可传播的一句话

--- 以下为生长层 ---

【作者笔记】写作时的真实状态和困惑
【生长日志】后续更新的记录
【留给你的问题】引导读者实践
```

> **写作时使用创作生长论版的模板。** 每章末尾加上生长层。

### 3.7 写作核心原则

- 先讲认知，再讲工具
- 国内工具先讲，国外工具作为进阶
- 每个工具必须说明"适合谁、解决什么、有什么缺点"
- 不要写成硬核教程
- 不要堆太多术语
- 不要离开中国普通人的真实使用场景
- 必须加入"从小白到做出来"的成长过程
- 必须保留作者的真实经历感
- 每章都要能拆成内容切片（用于抖音/公众号分发）
- **每章留 30% 的生长空间**，不一次写满

## 四、网站与部署

### 4.1 技术栈

| 层 | 技术 |
|---|------|
| 前端 | 原生 HTML + CSS + JS（单页应用，2386行） |
| 托管 | GitHub Pages |
| 域名 | https://r2fdm6ncsm-prog.github.io/InkGrowth-Notes/ |
| 自动部署 | GitHub Actions（.github/workflows/deploy.yml） |

### 4.2 部署方式

网站为**纯静态单页HTML**，位于 `prototype/index.html`。
根目录 `index.html` 和 `404.html` 均做 302 跳转到 `/prototype/index.html`。

**部署流程：**
1. 修改 `prototype/index.html` 或 `manuscript/` 中的内容
2. 将书稿 Markdown 手动更新到 `prototype/index.html` 中（当前为半自动流程）
3. 推送到 master 分支
4. GitHub Actions 自动部署到 GitHub Pages

### 4.3 内容更新流程

当前内容流转路径：
```
D:\A文件库\创作生长论\manuscript\*.md（书稿源文件）
        ↓ 人工/半自动转换
prototype/index.html（HTML渲染）
        ↓ git push
GitHub Pages（自动部署上线）
```

> ⚠️ **未来优化方向**：可引入静态站点生成器（如 MkDocs/Vitepress）实现 Markdown → HTML 自动转换。
